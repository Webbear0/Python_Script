校园二手教材循环交易系统 - 技术文档

1. 项目概况

本系统采用 Serverless 无服务器架构，基于 uni-app 跨平台框架和 uniCloud 云服务构建，包含移动端客户端、管理后台和官方网站三个子系统。

客户端使用 Vue 3 开发，支持 Android、iOS、H5 和微信小程序多端运行，提供教材发布与搜索、多维筛选（专业、年级、校区）、即时通讯、收藏管理等核心功能。管理后台基于 uni-admin 框架，集成 uni-id 统一身份认证，实现教材审核、用户管理、数据统计等管理功能。后端采用 uniCloud 云对象封装业务逻辑，使用 MongoDB 云数据库存储数据，通过 JQL 权限控制实现细粒度的数据访问控制。

  • 学生端 (Client)：面向学生的移动应用，包含 13 个页面，支持 Android / iOS / H5 / 微信小程序
  • 管理后台 (Server)：管理员使用的后台管理系统，采用 topWindow + leftWindow 经典布局
  • 官方网站 (Web)：纯静态 HTML 页面，项目介绍与下载

2. 技术架构

系统采用 Serverless 架构，基于 uniCloud (阿里云) 构建。Serverless 架构在运维成本、部署便捷性和弹性伸缩方面具有明显优势：云平台自动管理服务器，按调用量计费，支持自动弹性伸缩和一键部署，非常适合本系统这种校园级中小型应用。

2.1 技术栈

  • 前端框架: uni-app (Vue 3 Composition API, <script setup> 语法糖)
  • 管理后台: uni-admin (集成 uni-id 统一身份认证, RBAC 权限体系)
  • 后端服务: uniCloud 云对象 (campus-api 提供 19 个方法, admin-api)
  • 数据库: uniCloud Database (MongoDB 兼容, JQL 权限控制)
  • 云存储: uniCloud Storage (图片存储 + CDN 加速)
  • 用户认证: uni-id (注册、登录、Token 管理、角色权限 RBAC)

4. 学生端模块

客户端包含 13 个页面，采用底部 TabBar 导航设计，分为首页、发布和我的三个主模块。API 层按功能拆分为 auth.js（5 个方法）、textbook.js（6 个方法）、favorite.js（4 个方法）、message.js（4 个方法）四个独立模块。

4.1 首页与搜索

首页教材列表页是用户进入系统后的主界面，采用顶部搜索栏加底部分页列表的布局：

  • 关键词搜索：支持搜索 ISBN、书名、课程名
  • 多维筛选：按专业、年级、校区、发布类型（出售/求购）进行组合筛选
  • 分页加载：每页加载 10 条数据，滚动到底部自动加载下一页
  • 下拉刷新：支持手动刷新数据
  • 搜索历史：本地存储最近搜索记录

4.2 发布教材

支持出售和求购两种发布类型，填写教材名称、ISBN、价格、课程、专业、年级、校区、新旧程度等信息：

  • 图片上传：最多 4 张图片，自动压缩处理
  • 表单验证：必填项检查、价格范围验证
  • 编辑功能：支持修改已发布的教材信息
  • 状态管理：可以编辑、下架或删除已发布教材

4.3 个人中心

包含个人主页、我的发布、我的收藏、编辑资料、设置和关于 6 个页面：

  • 我的收藏：查看收藏的教材列表，支持取消收藏
  • 我发布的：管理自己发布的教材（上架/下架/删除）
  • 编辑资料：修改昵称、专业、年级、联系方式等个人信息
  • 设置：夜间模式切换、清除缓存
  • 关于页面：应用介绍和联系信息
  • 消息通知：显示未读消息数量

4.4 消息通讯

包括会话列表页和聊天页，支持一对一即时通讯：

  • 会话列表：显示所有聊天会话及未读数
  • 实时聊天：支持文字和图片两种消息类型
  • 用户信息：查看对方联系方式
  • 消息推送：新消息实时推送通知

5. 管理后台模块

管理后台基于 uni-admin 框架，采用 topWindow（顶部导航栏）和 leftWindow（左侧菜单栏）的经典布局结构。深度集成 uni-id 统一身份认证模块，通过 uniIdRouter 实现页面级别的自动登录拦截和权限校验。

5.0 仪表盘

  • 统计卡片：展示教材总数、在售教材、求购信息、注册用户、收藏次数和消息总数 6 项关键统计
  • 可视化图表：教材状态分布（待审核、已上架、已售出、已下架）柱状图
  • 快捷操作：快捷操作入口

5.1 用户管理

  • 用户列表：查看所有注册用户
  • 角色显示：支持 admin/student 等角色中文映射
  • 发布统计：显示每个用户发布的教材数量
  • 状态管理：启用/禁用用户账号
  • 权限控制：仅管理员可登录后台

5.2 教材管理

  • 教材审核：审核新发布的教材
  • 教材详情：查看教材完整信息
  • 状态变更：上架/下架/标记已售/删除教材

5.3 系统管理

  • 角色管理：配置系统角色
  • 权限管理：配置操作权限
  • 菜单管理：配置导航菜单
  • 安全日志：查看系统操作日志

6. 数据库设计

6.1 用户表 (uni-id-users)

  • username: 用户名/学号
  • nickname: 昵称
  • avatar: 头像URL
  • role: 角色数组 ['admin', 'student']
  • student_id: 学号
  • major, grade: 专业、年级
  • contact: 联系方式
  • reputation: 信誉积分（默认100）
  • register_date, last_login_date: 注册/登录时间

6.2 教材表 (textbooks)

  • title, isbn, author, course: 基本信息
  • price, original_price: 价格信息
  • condition, has_notes: 成色与笔记
  • grade, major, campus: 适用年级、专业、校区
  • description: 详细描述
  • status: 0-待审核, 1-已上架, 2-已售出, 3-已下架
  • images: 图片 URL 数组
  • publisher_id, publisher_name: 发布者 ID 与昵称
  • post_type: 'sell' 出售 / 'want' 求购
  • create_date, update_date: 发布/更新时间

6.3 收藏表 (favorites)

  • user_id: 用户 ID
  • textbook_id: 教材 ID
  • create_date: 收藏时间

6.4 消息表 (messages)

  • sender_id, receiver_id: 发送者/接收者
  • type: 'text' 文字 / 'image' 图片
  • content, image_url: 消息内容
  • is_read: 已读状态
  • created_at: 创建时间

7. 云函数接口

后端接口采用 uniCloud 云对象方式实现，客户端通过 uniCloud.importObject() 导入云对象实例后，可直接调用方法，无需手动编写 HTTP 请求。

7.1 campus-api（学生端，19 个方法）

用户认证（5 个方法）：

  • register: 用户注册 — 参数: username, password
  • login: 用户登录 — 参数: username, password
  • getUserInfo: 获取用户信息 — 参数: uid
  • updateUserInfo: 更新个人资料 — 参数: uid, data
  • setPushCid: 设置推送 ID — 参数: uid, push_clientid

教材管理（6 个方法）：

  • addTextbook: 发布教材 — 参数: title, price, uid...
  • getTextbooks: 获取教材列表 — 参数: keyword, page, pageSize
  • getTextbookById: 获取教材详情 — 参数: id
  • updateTextbook: 更新教材信息 — 参数: id, uid, data
  • updateTextbookStatus: 修改教材状态 — 参数: id, status, uid
  • deleteTextbook: 删除教材 — 参数: id, uid

收藏管理（4 个方法）：

  • addFavorite: 添加收藏 — 参数: uid, textbook_id
  • removeFavorite: 取消收藏 — 参数: uid, textbook_id
  • checkFavorite: 检查收藏状态 — 参数: uid, textbook_id
  • getFavorites: 获取收藏列表 — 参数: uid

即时消息（4 个方法）：

  • sendMessage: 发送消息 — 参数: uid, receiver_id, content
  • getConversations: 获取会话列表 — 参数: uid
  • getChatHistory: 获取聊天记录 — 参数: uid, other_id
  • getUnreadCount: 获取未读数 — 参数: uid

7.2 admin-api（管理端）

  • getTextbookCounts: 通过 MongoDB 聚合管道按发布者分组统计教材数量

8. 安全实现

  1. JQL 权限控制: 在数据库 Schema 中定义权限规则，实现声明式的细粒度权限控制（如：仅发布者本人或管理员可修改和删除教材）。
  2. 云函数校验: 关键业务逻辑在云端验证，对关键参数进行校验，防止非法请求和前端篡改。
  3. 管理员访问控制: 通过 uniIdRouter 配置页面级别自动登录拦截，非管理员角色禁止访问后台功能。
  4. 敏感字段过滤: 查询用户信息时自动过滤 password 等敏感字段。
  5. 密码加密存储: 用户密码加密存储，禁止明文保存。
  6. 图片压缩处理: 上传图片自动压缩，限制大小在 10MB 以内。
