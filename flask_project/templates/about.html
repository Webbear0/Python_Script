校园二手教材循环交易系统

基于 uni-app + uniCloud 的设计与实现——以广西财经学院为例

📖 项目背景

据教育部 2024 年统计公报，全国普通高校在校生已超过 4700 万人，每年消耗的教材数量相当可观，但大部分教材的实际使用周期只有一个学期。据调查，56.72% 的学生认为教材使用频率低，到大四阶段选择二手教材的学生比例高达 60.48%。

当前校园内二手教材的流通主要依赖 QQ 群、微信群等社交媒体群组以及拼多多、闲鱼等综合电商平台，存在信息分散、搜索困难、交易缺乏保障等突出问题。针对上述痛点，本项目设计并实现了一个面向高校学生的校园二手教材循环交易系统。

🎯 研究目的

本系统旨在解决当前校园二手教材流通中存在的核心问题，具体目标包括：

  • 精准搜索：构建结构化教材信息平台，支持按专业、年级、课程、校区等多维度精准筛选
  • 跨平台适配：一套代码适配 Android、iOS、H5 及微信小程序，降低用户使用门槛
  • 即时通讯：集成即时通讯功能，为买卖双方提供安全、便捷的沟通渠道
  • 后台管理：搭建管理后台，支持教材审核、用户管理和数据统计
  • 零运维部署：采用 Serverless 无服务器架构，实现低成本、高可用的部署方案

📝 研究意义

本研究的意义主要体现在两方面：

理论层面——目前国内关于 Serverless 架构在校园应用中的实践案例还不多，尤其是用 uniCloud 云对象替代传统 RESTful API 这种做法，本项目的开发过程可以作为一个参考。

实践层面——系统上线后能直接服务于校园教材的二次流通，让想买二手书的同学找得到书、想卖书的同学卖得出去。管理后台积累的教材流通数据，也能帮助学校在教材采购方面做出更合理的安排。

🏗️ 系统概述

本系统采用 Serverless 无服务器架构，基于 uni-app 跨平台框架和 uniCloud 云服务构建，包含三个子系统：

  • 移动端客户端：使用 Vue 3 开发，支持 Android、iOS、H5 和微信小程序，提供教材发布与搜索、多维筛选、即时通讯、收藏管理等核心功能
  • 管理后台：基于 uni-admin 框架，集成 uni-id 统一身份认证，实现教材审核、用户管理、数据统计等管理功能
  • 官方网站：项目介绍、技术文档与客户端下载页面
